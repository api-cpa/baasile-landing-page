Restriction d’accès à l’API et au dashboard

Dorénavant, tout accès non sécurisé à l’API et au dashboard généreront une erreur.
Nous vous invitons à mettre à jour vos favoris et les URI utilisées dans vos applications afin que vos utilisateurs ne soient pas impactés.

Erreur via le dashboard Filtrez les résultats d’une requête API Vous pouvez dorénavant filtrer les résultats de vos requêtes en utilisant le paramètre “filter”. Actuellement, cette fonctionnalité est uniquement implémentée sur les données des collections. Référez-vous à la documentation pour plus de détails. Exemple d’utilisation simple : ?filter[data.county]=75017 Se traduisant par : Toutes les données dont le code département est 75017. Exemple d’utilisation complexe : ?filter[$and][data.county]=75017&filter[$and][$or][data.street]=Bd+Bessières&filter[$and][$or][data.number]=96 Se traduisant par : Toutes les données dont le code du département est 75017 et dont l’une des deux conditions suivantes est remplie : Le nom de la rue est strictement égal à “Bd Bessières” ou le numéro de rue est strictement égal à “96” (ce qui n’a pas de sens, mais c’est pour l’exemple ;-)).
